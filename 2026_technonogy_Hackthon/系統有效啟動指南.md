## ---

**🚀 SDG 12 黑客松系統啟動全攻略**

建議當天提早 **15 分鐘** 到場，按照以下順序手動檢查與啟動。

### **第一階段：基礎設施檢查 (IoT 關鍵)**

1. **確認網路連線：**  
   * 確認你的電腦與 ESP32 連接在**同一個 Wi-Fi 熱點**下。  
   * 打開終端機輸入 ipconfig，確認並記下你的 **IPv4 位址** (如 192.168.1.100)。  
2. **啟動 MQTT 交換機 (Mosquitto)：**  
   * 因為我們已經設定為背景服務，通常它是開機即啟動。  
   * 若不放心，可在服務 (Services) 中手動點擊「重新啟動」。

### ---

**第二階段：四個終端機視窗 (核心啟動)**

在 VS Code 或 Anaconda Prompt 中開啟 **4 個不同的終端機分頁**，並確保每個分頁都先執行 conda activate hackthon 進入環境。

| 順序 | 執行指令 | 視窗功能描述 |
| :---- | :---- | :---- |
| **1** | uvicorn main:app --host 0.0.0.0 --port 8000 --reload | **API 伺服器：** 提供資料給 APP，並執行自動折價邏輯。 |
| **2** | streamlit run admin.py | **管理後台：** 供評審看即時數據與匯入測試 CSV。 |
| **3** | python mqtt_listener.py | **硬體監聽器：** 負責接收實體機台的購買訊號並扣庫存。 |
| **4** | .\mosquitto.exe -c mosquitto_2.conf -v

### ---

**第三階段：Demo 示範流程 (評審必看)**

當評審走到你們攤位前，請按照這套腳本展示：

1. **資料初始化 (顯示管理能力)：**  
   * 打開 Streamlit 後台 (localhost:8501)，現場上傳 test\_data.csv。  
   * **話術：** 「我們系統支援 CSV 一鍵匯入，能快速部署不同地點的機台資料。」  
2. **動態折價 Demo (亮點：責任消費)：**  
   * 在後台表格中指給評審看那些「即將到期」且「自動變為 5 折」的商品。  
   * 打開 API 網頁 (localhost:8000/api/machines/M001/products)。  
   * **話術：** 「後端 API 會自動運算剩食時間，主動提供優惠給前端 APP，實現 SDG 12 減少食物浪費。」  
3. **硬體連動 (亮點：軟硬整合)：**  
   * 請硬體隊友按下實體按鈕（或執行 test\_esp32.py）。  
   * 觀察 mqtt\_listener.py 終端機彈出「商品已售出」訊息，並重新整理後台，商品即時消失。  
   * **話術：** 「透過 MQTT 協定，實體機台的銷售會即時回傳資料庫，達成庫存自動化管理。」

### ---

**🛠️ 突發狀況排除 (Debug)**

* **ESP32 連不上 MQTT：**  
  * 檢查防火牆：請嘗試關閉 Windows Defender 防火牆 5 分鐘測試。  
  * 檢查 IP：確保 ESP32 程式碼裡的 IP 跟 ipconfig 查到的一模一樣。  
* **網頁 404 或資料沒出來：**  
  * 檢查 Uvicorn 是否噴紅字？  
  * 直接刪除 database.db，重開 Uvicorn 讓它重新生成乾淨的資料表。  
* **Streamlit 報錯 Table Defined：**  
  * 確認 main.py 裡有加上 \_\_table\_args\_\_ \= {'extend\_existing': True}。
