# SDG 12 責任消費系統 - 系統運作邏輯說明

本系統旨在透過物聯網 (IoT) 與後端自動化邏輯，達成減少剩食 (SDG 12.3) 的目標。以下為系統的詳細運作邏輯與架構說明。

---

## 一、 系統架構概覽 (System Architecture)

本系統由四個核心組件構成，協同完成資料採集、邏輯處理與資訊展示：

1.  **FastAPI 後端伺服器 (`main.py`)**
    *   **角色：** 系統的大腦與資料介面。
    *   **功能：** 提供 RESTful API 供前端 (APP/網頁) 查詢商品；執行關鍵的「動態折價邏輯」。
2.  **MQTT 監聽器 (`mqtt_listener.py`)**
    *   **角色：** 硬體與軟體的溝通橋樑。
    *   **功能：** 接收來自實體機台 (ESP32) 或模擬器的訊號，包含「商品售出」與「設備心跳」。
3.  **Streamlit 管理後台 (`admin.py`)**
    *   **角色：** 管理員與評審的操作介面。
    *   **功能：** 即時數據視覺化 (KPI、圖表、地圖)、商品 CRUD 管理、測試資料批次匯入。
4.  **SQLModel/SQLite 資料庫 (`database.db`)**
    *   **角色：** 系統的記憶體。
    *   **功能：** 儲存商品資訊 (Product) 與機台連線狀態 (MachineStatus)。

---

## 二、 核心業務邏輯 (Core Business Logic)

### 1. 動態折價機制 (Dynamic Discounting)
*   **觸發點：** 當任何終端透過 `/api/machines/{machine_id}/products` 請求商品列表時。
*   **運算邏輯：**
    *   系統會計算 `到期時間 (expiry_time)` 與 `當前時間` 的差距。
    *   **規則：** 若商品距離到期時間 **小於 2 小時且大於 0 小時**，且尚未標記為打折，則：
        *   將 `current_price` 修改為 `original_price` 的 **50% (五折)**。
        *   將 `is_discounted` 設為 `True`。
    *   **目的：** 透過價格誘導，優先出清即將過期的剩食，達成 SDG 12 目標。

### 2. 即時庫存管理 (Inventory Management)
*   **觸發點：** ESP32 偵測到使用者購買行為，發送 MQTT 訊號至主題 `sdg12/machine/sold`。
*   **動作：** `mqtt_listener.py` 接收到 JSON 格式 `{ "product_id": X }` 後，立即從資料庫中刪除該筆商品紀錄。
*   **同步：** 資料庫異動後，管理後台與 API 呼叫者將在下次更新時看到最新的庫存狀態。

### 3. 設備在線監測 (Heartbeat Mechanism)
*   **觸發點：** ESP32 定期發送心跳訊號至主題 `sdg12/machine/status`。
*   **動作：** `mqtt_listener.py` 更新 `MachineStatus` 資料表中的 `last_seen` 欄位。
*   **展示：** 管理後台會判斷 `last_seen` 是否在 30 秒內，若超過則顯示「設備已離線」，確保 Demo 過程中的硬體可靠性。

---

## 三、 資料流向 (Data Flow)

### 1. 購買流程
`實體機台 (按鈕/感測器)` -> `MQTT 訊息` -> `mqtt_listener.py` -> `刪除 SQLite 紀錄` -> `Admin/API 即時反映`

### 2. 查詢與折價流程
`前端 APP` -> `調用 FastAPI` -> `讀取資料庫` -> `執行折價運算` -> `寫回資料庫` -> `回傳 JSON 給前端`

### 3. 初始化流程
`管理員` -> `上傳 CSV 到 Streamlit` -> `寫入 SQLite` -> `系統開始運作`

---

## 四、 關鍵技術棧

*   **語言：** Python 3.10+
*   **Web 框架：** FastAPI (高效、非同步)
*   **資料庫 ORM：** SQLModel (結合 SQLAlchemy 與 Pydantic 的優點)
*   **即時通訊：** MQTT (Mosquitto Broker)
*   **資料視覺化：** Streamlit (快速構建管理面板)
*   **硬體端：** ESP32 (使用 WiFi 與 PubSubClient 庫)

---

## 五、 Demo 亮點說明

1.  **自動化：** 剩食折價不需人工調整，系統根據時間自動判斷。
2.  **低延遲：** 使用 MQTT 協定，購買後庫存更新近乎即時。
3.  **韌性：** 若硬體斷線，後台地圖會立刻變色警示，並可隨時切換至 `test_esp32.py` 進行軟體模擬。
4.  **視覺化：** 透過 Folium 地圖與 Plotly 圖表，清楚展示 SDG 12 的執行成果 (如：已節省的食物價值)。
